TD — 01

Physical Transmission

Bandwidth B

A medium acts as a filter — only frequencies in [Fmin, Fmax] pass through. Everything outside is heavily attenuated.

B = Fmax − Fmin

Phone line: 300–3400 Hz → B = 3100 Hz
Coaxial cable: B ≈ 300–400 MHz

Modulation Rate R

Number of symbols transmitted per second (in Bauds). One symbol can carry multiple bits.

R = 1 / T

T = duration of one symbol (seconds)
Max rate: Rmax = 2 × Fmax

Valence V

Number of distinct states a signal can take. Binary signal: V=2. Quadrivalent: V=4. More states = more bits per symbol.

log₂(V) = bits per symbol

V=2 → 1 bit/symbol
V=4 → 2 bits/symbol
V=16 → 4 bits/symbol

Bit Rate D

Number of bits transmitted per second. Connects modulation rate to actual data throughput.

D = R × log₂(V)

If V=2: D = R (same as baud rate)
If V=4: D = R × 2

Attenuation

Signal loses energy over distance. Measured as the ratio of output power to input power, expressed in decibels.

A(dB) = 10 × log₁₀(S/N)

Error Rate

Probability that a bit is lost or corrupted during transmission. Measured as the ratio of erroneous bits to total bits sent over a significant time period.

τ = nb_bits_erronés / nb_bits_émis
1.1

Nyquist Theorem

For a noiseless channel. Gives the theoretical maximum bit rate based on bandwidth and signal valence.

Nyquist — Noiseless Channel
Dmax = 2 × F × log₂(V) F = bandwidth (Hz) · V = valence · Dmax in bits/s
▶ Example — Exercise 1

D = 2400 bit/s, V = 2 (bivalent) → R = D / log₂(2) = 2400 / 1 = 2400 bauds

Same D, V = 4 (quadrivalent) → R = 2400 / log₂(4) = 2400 / 2 = 1200 bauds

▶ Example — Exercise 3

D = 9600 bit/s, V = 16 (4 bits per symbol) → log₂(16) = 4

From Nyquist: F = Dmax / (2 × log₂(V)) = 9600 / (2 × 4) = 1200 Hz

1.2

Shannon Theorem

For a noisy channel. Absolute upper bound — no coding scheme can exceed this.

Shannon — Noisy Channel
C = F × log₂(1 + S/N) S/N is a power ratio — NOT in dB. Convert first if needed.
⚠ dB Conversion: If SNR is given in dB, convert: S/N = 10^(dB / 10)
Example: 20 dB → S/N = 10^(20/10) = 10² = 100
▶ Example — Exercise 5

F = 3000 Hz, SNR = 20 dB → S/N = 100

C = 3000 × log₂(1 + 100) = 3000 × log₂(101)

log₂(101) ≈ 6.658 → C ≈ 3000 × 6.658 = ≈ 19,974 bit/s

▶ Example — Exercise 4

Cat5 twisted pair: F = 100 MHz, S/N = 100 (signal is 100× noise)

C = 100×10⁶ × log₂(101) ≈ 100×10⁶ × 6.658 ≈ 665.8 Mbit/s

1.3

Transmission vs Propagation Delay

Critical distinction: These are two completely different things. The exam will mix them — don't.

Transmission Time Tt

Time to push all bits of a message onto the wire. Depends on message size and bit rate.

Tt = Size (bits) / D (bit/s)

Propagation Delay Tp

Time for a single bit to travel from one end to the other. Depends on cable length and propagation speed.

Tp = L / V
Total end-to-end delay
T_total = Tt + Tp = (Size / D) + (L / V)
▶ Example — Exercise 6 (Optical fiber)

D = 155 Mb/s, L = 3000 km = 3×10⁶ m, V = 2×10⁸ m/s, packet = 512 octets = 4096 bits

Tt = 4096 / (155×10⁶) ≈ 26.4 μs

Tp = 3×10⁶ / (2×10⁸) = 15 ms

Total ≈ 15.026 ms (propagation dominates)

▶ Same packet, twisted pair (D = 2 Mb/s)

Tt = 4096 / (2×10⁶) = 2.048 ms

Tp = same = 15 ms

Total ≈ 17.05 ms — transmission time is now significant

Manchester

Always a transition at the middle of each bit period.

0 → ↑ (low-to-high at mid)  |  1 → ↓ (high-to-low at mid)

Rule

Bit 1: starts HIGH, transitions to LOW at midpoint → ↓

Bit 0: starts LOW, transitions to HIGH at midpoint → ↑

Self-clocking: the mid-bit transition is always present, so the receiver can always find the clock.

Differential Manchester

Mid-bit transition always exists (for clocking). Data encoded by start-of-bit transition.

Bit 1

No transition at the start of the bit period. Only the mid-bit transition occurs.

Bit 0

Transition at the start of the bit period, plus the mid-bit transition.

Exam tip: When decoding from a diagram, identify the mid-point of each cell. In Manchester, look at the direction of the mid-transition (↓=1, ↑=0). In Differential Manchester, look at whether there's a transition at the cell boundary (transition=0, no transition=1).

Miller (Delay Modulation)

Bit 1

Transition at the middle of the bit period.

Bit 0 (isolated)

No transition at middle or start — level stays constant.

Bit 0 (after 0)

Transition at the start of the bit period (boundary between two zeros).

Odd parity (impaire): Add a bit so total number of 1s in the word is odd.

▶ Exercise 4 — "NET" with odd parity

N = 1001110 → four 1s (even) → parity bit = 1 → transmitted: 10011101

E = 1001001 → three 1s (odd) → parity bit = 0 → transmitted: 10010010

T = 1010100 → three 1s (odd) → parity bit = 0 → transmitted: 10101000

2.3

CRC — Cyclic Redundancy Check

1
Identify degree r of the generator polynomial g(x). That's the highest power of x.
2
Append r zeros to the original message M → this is M × 2^r.
3
XOR divide M×2^r by the generator G (binary long division, no borrows — only XOR).
4
Remainder R (r bits) is the CRC. Transmitted message = M + R (concatenated).
Verification: Divide received message by G. If remainder = 0 → no error detected. If ≠ 0 → error.
▶ Exercise 5 — CRC Encoding

Message M = 0011111101, g(x) = x² + x + 1 → G = 111, degree r = 2

Append 2 zeros: M' = 001111110100

XOR divide 001111110100 by 111:

0011 ÷ 111 → 011 XOR 111 = 100 ... bring down bits one at a time
Continue until remainder has < 3 bits → remainder = 01

Transmitted: 001111110101 (append remainder 01)

▶ Exercise 5 — CRC Verification

Received = 101011000110, g(x) = x⁶+x⁴+x+1 → G = 1010011

Divide 101011000110 by 1010011 using XOR division.

If remainder = 0 → transmission correct. The message (without last 6 bits CRC) is the original data.

TD — 03

Hamming Code

Code Parameters

n = total bits (data + parity)
k = data bits
m = parity bits = n − k
dmin = 3 (always for standard Hamming)

Key Formula

For m parity bits:

n = 2ᵐ − 1
k = 2ᵐ − 1 − m

m=3: n=7, k=4 → code(7,4,3)
m=4: n=15, k=11 → code(15,11,4)

Error Capability

eD = dmin − 1 = 2
eC = ⌊(dmin−1)/2⌋ = 1

Standard Hamming: detects up to 2 errors, corrects 1 error.

3.1

Encoding — code(7,4)

Parity bits go at positions that are powers of 2. Data bits fill the rest. Positions are numbered right-to-left starting at 1.

Position 7654321
Type D7D6D5 P4 D3 P2P1

Each parity bit covers positions where the corresponding bit of the position index is 1:

P1 (pos 1 = 001₂)

Covers positions: 1, 3, 5, 7
(positions where bit 0 = 1)

Set P1 so that count of 1s in {P1, D3, D5, D7} is even

P2 (pos 2 = 010₂)

Covers positions: 2, 3, 6, 7
(positions where bit 1 = 1)

Set P2 so that count of 1s in {P2, D3, D6, D7} is even

P4 (pos 4 = 100₂)

Covers positions: 4, 5, 6, 7
(positions where bit 2 = 1)

Set P4 so that count of 1s in {P4, D5, D6, D7} is even

▶ Exercise 01 — Encode '0001'

Data bits: D7=0, D6=0, D5=0, D3=1

Place in grid: _ _ 0 _ ? 1 ? ? ? → positions 7,6,5 = 0,0,0 and pos 3 = 1

P1: covers {1,3,5,7} = {P1, 1, 0, 0} → one 1 → P1 = 1 (to make even)

P2: covers {2,3,6,7} = {P2, 1, 0, 0} → one 1 → P2 = 1

P4: covers {4,5,6,7} = {P4, 0, 0, 0} → zero 1s → P4 = 0

Final code: D7 D6 D5 P4 D3 P2 P1 = 0 0 0 0 1 1 1 → 0000111

3.2

Error Detection — Syndrome

Recompute all parity checks on the received word. Each failed check contributes its bit position to the syndrome. The syndrome value = position of the error.

1
Recompute P1, P2, P4 on the received word using the same coverage rules.
2
Build syndrome: if P4 fails → bit 2 = 1. If P2 fails → bit 1 = 1. If P1 fails → bit 0 = 1.
3
Syndrome value (in decimal) = position of the erroneous bit. Flip that bit.
▶ Exercise 02 — Verify '1011100'

Positions: P1=0, P2=0, D3=1, P4=1, D5=1, D6=0, D7=1

P1 check {1,3,5,7}: bits = 0,1,1,1 → three 1s → odd → FAIL (C1=1)

P2 check {2,3,6,7}: bits = 0,1,0,1 → two 1s → even → OK (C2=0)

P4 check {4,5,6,7}: bits = 1,1,0,1 → three 1s → odd → FAIL (C4=1)

Syndrome = C4 C2 C1 = 1 0 1 = 5 → error at position 5 (D5)

Received: 1011100 → flip pos 5 → corrected: 1010100 = 1010100
